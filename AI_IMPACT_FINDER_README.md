# AI Impact Finder - Complete Guide

## Overview

The AI Impact Finder helps business owners identify high-value AI opportunities through transparent, heuristic-based scoring. It discourages poorly-implemented customer-facing AI while highlighting safe, effective back-office improvements.

## How It Works

### Scoring Algorithm

```
For each AI use case:

1. Calculate Opportunity Score
   opportunityScore = normalize(baseImpact + sum(signals))

2. Apply Penalties
   trustPenalty = customerTrustRisk
   maturityPenalty = max(0, (maturityRequired - userCapability) * 2)

3. Calculate Net Value
   netValue = opportunityScore - trustPenalty - maturityPenalty

4. Determine Confidence
   signals >= 3: High confidence
   signals == 2: Medium confidence
   signals <= 1: Low confidence

5. Apply Gating Rules
   Suppress customer-facing AI if maturityRequired > userCapability
```

### Results Format

**Section A: Summary** - How results were determined
**Section B: Top 2 Recommendations** - Best opportunities with explanations
**Section C: Use Cautiously** - Suppressed or high-risk items (required)
**Section D: Philosophy** - Statement on customer-facing AI
**Section E: CTA** - Low-pressure contact option


### Add/Remove Use Cases

Edit [data/ai-use-cases.json](data/ai-use-cases.json):

```json
{
  "id": "new-use-case",
  "title": "Your Use Case",
  "description": "...",
  "baseImpact": 7,
  "customerTrustRisk": 2,
  "maturityRequired": 3,
  "customerFacing": false,
  "requiresOptIn": false,
  "examples": ["Example 1", "Example 2"]
}
```

Then add signals in [data/questionnaire.json](data/questionnaire.json).

### Modify Questions

Edit [data/questionnaire.json](data/questionnaire.json):

```json
{
  "id": "your-question",
  "question": "Your question text?",
  "options": [
    {
      "id": "option-1",
      "label": "Option label",
      "signals": {
        "use-case-id": 2
      }
    }
  ]
}
```

### Adjust Scoring

Edit [lib/ai-impact-scoring.ts](lib/ai-impact-scoring.ts):

- Normalization range (line 148)
- Maturity penalty multiplier (line 157)
- Confidence thresholds (line 222)

## Design Principles

1. **Transparency** - Every score is explainable
2. **Safety** - Gatekeeping for risky customer-facing AI
3. **Clarity** - Qualitative labels over fake precision
4. **Honesty** - Shows what won't work, not just what will
5. **Utility** - Results can be saved and shared
